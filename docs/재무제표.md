재무제표(Financial Statements)를 뽑는 과정은 개발자 입장에서 보면 **"거대한 원장(Ledger) 테이블을 특정 기준으로 Group By 하고 합산(Sum)하는 집계 작업"**입니다.

하지만 단순히 SQL만 날린다고 끝나는 게 아니라, 회계 특유의 **'마감(Closing)'**이라는 절차가 핵심입니다. 고성능 시스템 설계를 선호하시는 사용자님의 스타일에 맞춰 공학적인 관점에서 설명해 드릴게요.

---

## 1. 보고서 생성을 위한 데이터 소스

보고서를 뽑을 때 `Journal_Lines`(전표 상세)를 매번 처음부터 다 읽으면 성능이 터집니다. 그래서 보통 두 가지 루트를 탑니다.

* **전통적 방식:** `Account_Balances`(잔액 테이블)를 사용합니다. 월 단위나 일 단위로 미리 합산된 값을 읽어오기 때문에 매우 빠릅니다.
* **현대적 방식 (HANA/ACDOCA 스타일):** 인메모리 DB를 사용해 `Journal_Lines`급의 대용량 테이블을 실시간으로 집계(On-the-fly Aggregation)합니다.

---

## 2. 재무제표별 집계 로직 (The Logic)

회계 보고서는 크게 두 가지 성격으로 나뉩니다.

### ① 재무상태표 (Balance Sheet, BS) - "특정 시점의 스냅샷"
* **공식:** **자산 = 부채 + 자본**
* **집계 방법:** 회사 설립일부터 **조회하려는 시점까지의 모든 누적 잔액**을 합산합니다.
* **데이터 예시:** "현재 우리 금고에 총 얼마 있어?"를 묻는 것이므로, 과거의 모든 입출금을 다 더해야 합니다.

### ② 손익계산서 (Income Statement, IS) - "특정 기간의 성과"
* **공식:** **수익 - 비용 = 당기순이익**
* **집계 방법:** **조회하려는 기간(예: 1월 1일 ~ 3월 31일)** 사이의 거래만 합산합니다.
* **데이터 예시:** "이번 분기에 보험료로 얼마 벌었고, 월급으로 얼마 나갔어?"를 묻는 것이므로, 기간 외 데이터는 버립니다.

---

## 3. 보고서 추출 3단계 프로세스

### STEP 1: 결산 및 마감 (Closing)
데이터를 뽑기 전에 "이제 더 이상 이 기간의 데이터는 수정 못 해!"라고 자물쇠를 거는 작업입니다.
* **자동 분개:** 감가상각비, 미수금 등을 시스템이 자동으로 계산해서 전표를 추가합니다.
* **상태 변경:** 해당 월의 전표 상태를 `Confirmed`에서 `Closed`로 변경하여 무결성을 확보합니다.

### STEP 2: 계정 매핑 (Reporting Mapping)
원장의 계정과목은 수천 개일 수 있지만, 보고서에는 "현금 및 현금성 자산"처럼 묶어서 보여줘야 합니다.
* **Mapping Table:** `Account_Code` 10101, 10102, 10103은 모두 보고서의 `Cash_and_Cash_Equivalents` 라인에 합산하라는 규칙을 적용합니다.

### STEP 3: 집계 SQL (Conceptual)
개발자에게 가장 익숙한 코드 형태로 보면 이렇습니다.

```sql
-- 손익계산서(IS) 예시: 특정 기간의 수익/비용 합산
SELECT 
    m.Report_Line_Name,  -- 보고서에 표시될 이름 (매핑 테이블 조인)
    SUM(l.Debit_Amount - l.Credit_Amount) AS Net_Balance
FROM Journal_Lines l
JOIN Account_Master a ON l.Account_Code = a.Account_Code
JOIN Report_Mapping m ON a.Account_Code = m.Account_Code
WHERE l.Post_Date BETWEEN '2026-01-01' AND '2026-03-31'
  AND a.Account_Type IN ('Revenue', 'Expense') -- 수익과 비용만
GROUP BY m.Report_Line_Name;
```

## 4. 고성능 시스템을 위한 아키텍처 제언

사용자님이 보험사나 증권 시스템을 만든다면, 수억 건의 데이터를 처리해야 하므로 다음 기법을 고려해보세요.

* **Materialized View (실시간 집계 뷰):** 전표가 생성될 때마다 트리거나 이벤트 스트림(Kafka)을 통해 보고서용 집계 테이블을 실시간으로 업데이트합니다. 사용자가 보고서를 누르는 순간 0.1초 만에 결과가 나옵니다.
* **OLAP Cube:** 데이터를 미리 다차원(시간, 부서, 계정)으로 인덱싱해두어, 사용자가 드릴다운(Drill-down, 합계 클릭 시 상세 내역 보기)을 할 때 지연 시간이 없게 만듭니다.

## 💡 요약하자면

재무제표는 **"정해진 기간의 데이터를 회계 규칙에 따라 합쳐서 예쁘게 출력하는 것"**입니다.

1.  **잔액 테이블**에서 데이터를 땡겨오고,
2.  **보고서 양식(매핑)**에 맞춰 숫자를 끼워 넣고,
3.  **차변/대변 합계가 일치하는지** 최종 검증(Validation)한 뒤 화면에 뿌려줍니다.
